# Панель менеджера дзвінків

Невеликий вебзастосунок на **Streamlit**, який виступає єдиною стартовою панеллю для менеджера дзвінків. Застосунок показує три великі картки з коротким описом і кнопкою переходу до відповідного робочого інструмента.

## Що робить проєкт

- Відкриває зручну **панель навігації** для щоденної роботи менеджера.
- Об’єднує в одному інтерфейсі 3 зовнішні інструменти:
  - **Звіт за день** — формування денного звіту менеджера.
  - **Розподіл заявок** — автоматичний розподіл заявок за напрямком.
  - **Кошторис** — підготовка прорахунку для клієнта.
- Відкриває кожен інструмент у **новій вкладці браузера**.
- Має широке компонування (`layout="wide"`) для комфортної роботи з десктопа.

## Структура проєкту

```text
.
├── app.py      # Основний Streamlit-застосунок
└── README.md   # Документація проєкту
```

## Технології

- **Python 3.10+** (рекомендовано)
- **Streamlit**
- HTML/CSS-вставки через `st.markdown(..., unsafe_allow_html=True)` для кастомних карток і кнопок

## Швидкий старт

### 1) Клонування репозиторію

```bash
git clone <URL_ВАШОГО_РЕПОЗИТОРІЮ>
cd panel_for_manager_call
```

### 2) Створення віртуального середовища (рекомендовано)

```bash
python -m venv .venv
source .venv/bin/activate   # Linux/macOS
# .venv\Scripts\activate    # Windows (PowerShell)
```

### 3) Встановлення залежностей

Оскільки в репозиторії наразі немає `requirements.txt`, можна встановити мінімальну залежність вручну:

```bash
pip install streamlit
```

> За потреби ви можете зафіксувати версії в `requirements.txt` для стабільних деплоїв.

### 4) Запуск локально

```bash
streamlit run app.py
```

Після запуску Streamlit виведе локальний URL (зазвичай `http://localhost:8501`), де буде доступна панель.

## Налаштування та кастомізація

### Зміна URL інструментів

У файлі `app.py` посилання зібрані в окремі константи:

- `TOOL_1_URL`
- `TOOL_2_URL`
- `TOOL_3_URL`

Для оновлення переходів достатньо змінити ці змінні в одному місці.

### Додавання нового інструмента

Список карток будується зі структури `TOOLS` (список словників). Щоб додати ще один сервіс:

1. Додайте новий словник у `TOOLS` з полями:
   - `name` — назва,
   - `description` — короткий опис,
   - `emoji` — іконка,
   - `url` — посилання.
2. Переконайтесь, що URL валідний і доступний.
3. Перезапустіть застосунок.

Кількість колонок формується автоматично через `st.columns(len(TOOLS), gap="large")`.

### Стилізація інтерфейсу

Кастомні стилі задаються блоком CSS у `st.markdown(...)`:

- `.tool-card` — зовнішній вигляд картки,
- `.tool-title` — заголовок,
- `.tool-description` — текст опису,
- `.tool-link` — кнопка переходу.

За потреби можна змінити кольори, відступи, тіні, радіус кутів і розміри шрифтів.

## Типовий сценарій використання

1. Менеджер відкриває панель.
2. Обирає потрібний інструмент залежно від задачі.
3. Натискає кнопку **«Відкрити інструмент»**.
4. Сервіс відкривається в новій вкладці, а панель лишається як «домашня» сторінка навігації.

## Можливі покращення

- Додати `requirements.txt` або `pyproject.toml`.
- Додати перевірку доступності зовнішніх URL.
- Винести конфігурацію інструментів у JSON/YAML.
- Додати ролі/права доступу, якщо панель використовують різні групи.
- Підключити базову аналітику кліків по інструментах.
